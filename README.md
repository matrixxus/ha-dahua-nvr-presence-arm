# Dahua NVR Presence Arm/Disarm for Home Assistant

Automatically arm and disarm your Dahua NVR based on household presence, with a night-mode override that keeps cameras armed while you sleep.

## What it does

- **Arms** the NVR when everyone leaves home
- **Disarms** when someone arrives home (only outside night hours)
- **Night mode** — arms at a configurable time every evening, even if you're home
- **Morning disarm** — disarms at a configurable time if someone is home
- **Manual override** — manually arming via the switch persists until an automation trigger changes it

This works by toggling the Dahua `DisableLinkage` setting via the NVR's CGI API. When linkage is enabled, the NVR processes alarm events and sends push notifications (e.g., via DMSS). When disabled, the NVR keeps recording but won't push notifications to your phone.

## Requirements

- Dahua NVR with CGI API access (tested on models with `configManager.cgi` support)
- Home Assistant 2024.x+
- [HA Companion App](https://companion.home-assistant.io/) for presence tracking (or any device tracker)
- [Card Mod](https://github.com/thomasloven/lovelace-card-mod) *(optional, for dashboard styling)*
- [Advanced Camera Card](https://github.com/dermotduffy/advanced-camera-card) *(optional, for camera feed dashboard)*

## Installation

### Step 1: Package file (entities)

1. Copy [`packages/dahua_nvr.yaml`](packages/dahua_nvr.yaml) to your HA `/config/packages/` directory
2. Add to your `secrets.yaml`:
   ```yaml
   dahua_user: "admin"
   dahua_password: "your_nvr_password"
   ```
3. Edit the package file:
   - Replace `YOUR_NVR_IP` with your NVR's IP address (3 places)
   - Replace `person.CHANGE_ME` with your actual person entity
4. Restart Home Assistant
5. Verify entities exist in **Developer Tools → States**:
   - `sensor.dahua_nvr_disarm_state` — should show `armed` or `disarmed`
   - The cameras switch — search for `dahua_nvr_armed` (entity ID is generated by HA, likely `switch.cameras_armed`)

### Step 2: Blueprint (automations)

[![Import Blueprint](https://my.home-assistant.io/badges/blueprint_import.svg)](https://my.home-assistant.io/redirect/blueprint_import/?blueprint_url=https://github.com/matrixxus/ha-dahua-nvr-presence-arm/blob/main/blueprints/automation/dahua_nvr_presence_arm.yaml)

Or manually:
1. Copy [`blueprints/automation/dahua_nvr_presence_arm.yaml`](blueprints/automation/dahua_nvr_presence_arm.yaml) to `/config/blueprints/automation/dahua_nvr/`
2. Go to **Developer Tools → YAML → Reload Automations**
3. Go to **Settings → Automations → Blueprints** → find "Dahua NVR Arm/Disarm" → **Create Automation**
4. Configure:
   - **Cameras Armed Switch** — select your cameras switch entity
   - **Presence Group** — select `group.household` (or a person entity)
   - **Night Mode Start** — default 21:00
   - **Morning Disarm Time** — default 07:00
5. Save

## How the logic works

| Time | Event | Result |
|---|---|---|
| 07:00–21:00 | Everyone leaves | Arm |
| 07:00–21:00 | Someone arrives | Disarm |
| 07:00–21:00 | Manual arm toggle | Stays until next trigger |
| 21:00 | Night mode starts | Arm (regardless of presence) |
| 21:00–07:00 | Someone arrives | Stays armed (night mode active) |
| 07:00 | Morning + someone home | Disarm |
| 07:00 | Morning + nobody home | Stay armed |

## Dahua compatibility

This uses the standard Dahua CGI API endpoint:

```
http://<NVR_IP>/cgi-bin/configManager.cgi?action=getConfig&name=DisableLinkage
```

Expected response format:
```
table.DisableLinkage.Enable=false
```

Where `false` = linkage enabled (armed) and `true` = linkage disabled (disarmed).

Tested with Dahua NVR models supporting `configManager.cgi`. If your NVR returns a different response format, you may need to adjust the `value_template` in the REST sensor.
